# 자녀 교육→대입 파이프라인 (edu-pipeline) 기획안 v2

- 작성: 2026-07-07 · 3AI 교차검증 기반(v1) + 진로진학 통합(v2)
- 근거: `~/.claude/skills/3ai/outputs/edu-admission-pipeline_synthesis.md`

## 제품 정의

가족용 **장기 로드맵 + 근거자료 저장소 + 진로·지원전략 도구**.
상용 서비스가 아닌 개인/가족용. '어디가' 등 공식 도구가 이미 하는 기능(성적분석·합격진단)은 복제하지 않고, **가족의 장기 기록과 의사결정 맥락**을 담당한다.

## 핵심 관점: 진로가 상류다

```
진로 탐색(흥미·적성) → 전공 가설 → 고교/과목 선택 → 전형 선택 → 지원 실행
        (초~중)          (중~고1)      (중3~고2)      (고2~고3)    (고3)
```

대입은 마지막 단계일 뿐, 그 앞의 모든 선택(고교 유형, 선택과목, 활동 방향)이 진로 가설에서 갈라진다. 따라서 **진로 모듈이 파이프라인의 시작점**이며, 진로 가설이 바뀌면 하류(과목·전형 계획)에 "재검토 필요" 플래그가 걸리는 구조로 설계한다.

## 정보 구조

```
홈 — 이번 달 할 일 · D-day · 재확인 필요 정보
├─ 진로·전공 탐색 ★v2 신설
│  ├─ 진로 가설 보드      현재 가설 + 변경 이력(변경은 정상 — 탐색의 증거)
│  ├─ 흥미·적성 검사 기록  커리어넷/워크넷 검사 결과, 시기별 변화
│  ├─ 직업·학과 카드      조사한 직업/학과 정보 + 출처 + 자녀 소감
│  ├─ 체험·탐색 활동      진로체험, 직업인 인터뷰, 전공 관련 독서/탐구
│  └─ 전공↔과목 연계표    희망 전공별 권장과목 vs 학교 개설과목 vs 이수 계획
├─ 성장 기록
│  ├─ 학습 습관·과목별 상태
│  ├─ 활동·독서·탐구·성찰
│  └─ 상담·의사결정 기록 (학교 진로상담 포함)
├─ 고교 교육과정
│  ├─ 고교 유형 비교(중3용) · 과목 선택 · 이수 현황
├─ 대입 정보
│  ├─ 대학/학과 · 전형·지원자격 · 수능최저
│  └─ 모집요강 원문 + 변경 이력 ← 메타데이터 강제
├─ 지원 전략
│  ├─ 관심/검토/지원 대학 · 상향/적정/안정 시나리오
│  └─ 수시·정시 일정, 제출 체크 (충족/미충족/확인 필요 — 확률 숫자 금지)
└─ 설정 — 개인정보·백업·공유(자녀가 공개 범위 결정)
```

## 단계별 로드맵 (진로진학 통합)

| 단계 | 교과·입시 축 | 진로진학 축 ★ |
|---|---|---|
| 초등 | 학습 습관, 독서 | 흥미 발견 — 좋아하는 주제·질문 기록. 검사·직업 조사 강요 금지 |
| 중1~2 | 내신 기반, 자기주도학습 | 넓은 탐색 — 커리어넷 검사, 자유학기제 체험, 직업 카드 수집. 진로 고정 금지 |
| 중3 | 고교 선택·학습 격차 점검 | 진로 가설 1차 정리 → 고교 유형(일반/특목/자사/특성화) 선택의 근거로 |
| 고1 | 내신·교과 탐구 | 계열 가설(인문/사회/자연/공학/의약 등) + 선택과목 1차 설계 |
| 고2 | 과목 이수, 활동의 근거 축적 | 전공 가설 구체화 → 전공↔과목 연계표 확정, 대학별 요구사항 조사 시작 |
| 고3 | 지원 실행 | 학과─전형 매칭(같은 학과라도 대학별 전형·최저 상이), 면접 대비 |

각 단계의 진로 활동도 성장 기록과 같은 원칙: **사실(검사 결과)/해석(부모·교사 의견)/가설(희망 진로)을 분리 저장**.

## 진로진학 1차 출처 (공식 우선)

| 출처 | 용도 |
|---|---|
| 커리어넷 (career.go.kr) | 직업적성·흥미검사, 직업/학과 정보, 진로상담 — 교육부 운영 |
| 워크넷 (work.go.kr) | 직업심리검사, 직업 전망 — 고용노동부 |
| 꿈길 (ggoomgil.go.kr) | 진로체험처 검색 — 자유학기제 연계 |
| 어디가 (adiga.kr) | 학과 정보, 대학별 전형 — 대입 직결 |
| 각 대학 '전공 안내서'·권장과목 발표 | 전공↔과목 연계표의 근거 |
| 메이저맵 등 민간 서비스 | 2차 출처로 분리 태깅 |

## 설계 원칙 (v1 유지)

1. **메타데이터 강제**: 모든 입시·진로 정보에 적용 학년도·출처 URL·확인일·공식/예고/해석 구분.
2. **신선도 경고**: 90일 미확인 경고, 전년도 자료 사용 경고, 지원 직전 원문 재확인 체크.
3. **허위 정밀성 금지**: 합불 확률 숫자 대신 충족/미충족/확인 필요.
4. **감시 도구화 방지**: 자녀가 기록 공개 범위 결정. 진로 가설 변경을 "일관성 부족"으로 채점하지 않기.
5. **개인정보**: 성적·학생부 로컬 저장, 최소수집, 학생부 원문 외부 AI 입력 금지.
