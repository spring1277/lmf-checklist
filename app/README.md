# LMF 인증심사 점검표 준비 앱

진단검사의학재단(LMF) **우수검사실 신임인증** 연차별 지침서(준비자료) 작성을 돕는 **모바일 PWA**입니다.
분야별 심사점검표를 보고, 각 점검 기준에 대한 우리 검사실의 **답안(자기검증 근거)**을 작성·저장하며, AI 초안 작성을 붙일 수 있습니다.

---

## 1. 실행 방법

### PC에서
- `실행하기.bat` 더블클릭 → 브라우저에서 **http://127.0.0.1:8765** 접속
- (또는 터미널에서) `python -m http.server 8765` 실행 후 같은 주소 접속

> ⚠️ `index.html`을 파일로 직접 열면(`file://`) 데이터 로딩이 막힙니다. **반드시 로컬 서버로 실행**하세요.

### 휴대폰에서 (모바일 앱처럼 사용)
1. PC와 휴대폰을 **같은 Wi-Fi**에 연결
2. PC에서 `실행하기.bat` 실행 → 창에 표시된 `http://192.168.x.x:8765` 주소 확인
3. 휴대폰 브라우저(Chrome/Safari)에 그 주소 입력
4. 브라우저 메뉴 → **홈 화면에 추가** → 아이콘으로 앱처럼 실행 (오프라인 동작)

> 외부에서도 쓰려면 이 `app/` 폴더를 정적 호스팅(Netlify, GitHub Pages, Vercel 등)에 올리면 됩니다. HTTPS면 PWA 설치가 더 매끄럽습니다.

## 2. 기능

- **연도 전환**: 헤더 우측 2026 / 2025
- **공통분야 4종**(검사실운영·종합검증·현장검사·수탁검사, 2026): 기준별 **구조화 점검표**
  - 영역/소분류 접기·펼치기, 필터(전체·미작성·완료·핵심·필요·기본)
  - 기준코드 / 유형 배지(핵심 C·필요 R·기본 B) / 배점 / 기준 설명 토글
  - 답안 입력(자동 저장), 완료 체크, 진행률 표시
  - **🤖 AI 초안**: 공식 기준 설명을 점검 포인트로 분해한 **템플릿 초안** 자동 생성 (LLM 없이 동작)
- **그 외 전문분야 10종 + 2025년 전체**: **공식 PDF 원문 보기** + 분야 준비 메모
- **메뉴(⋮)**: 답안 내보내기(JSON) / 답안 보고서(인쇄·PDF 저장) / 불러오기 / AI 설정 / 연도 초기화
- 모든 답안은 휴대폰·PC의 **브라우저 내 IndexedDB**에 저장(서버 전송 없음). 기기 이전은 JSON 내보내기/불러오기로.

## 3. 폴더 구조

```
app/
├─ index.html, css/style.css, js/app.js, js/db.js   # 앱 본체
├─ manifest.webmanifest, sw.js, icons/              # PWA (설치·오프라인)
├─ data/
│  ├─ index.json                                    # 분야·연도 카탈로그
│  ├─ checklists/2026_*.json                         # 구조화 점검표(공통 4분야)
│  └─ pdf/{2025,2026}/*.pdf                          # 공식 점검표 PDF 28개
├─ 실행하기.bat
└─ README.md
```

원본 다운로드는 `../data/checklists/` 에 보관(2025·2026 × 14분야), 구조화 추출 결과는 `../data/checklists/extracted/` 에 있습니다.

## 4. AI 자동 초안 백엔드 연동 (선택, 추후)

기본은 **오프라인 템플릿 초안**입니다. 진짜 AI 자동 작성을 붙이려면:

1. 앱 메뉴 → **🤖 AI 초안 설정** → 엔드포인트 URL / API 키 / 모델 / 기관정보 입력·저장
2. 백엔드는 아래 계약(contract)을 구현하면 됩니다.

**요청 (앱 → 백엔드, POST JSON)**
```json
{
  "year": "2026", "field": "검사실운영",
  "code": "01.010.020", "type": "C", "points": null,
  "question": "점검 문항 전체",
  "guidance": "기준 설명 전체",
  "labContext": "기관/검사실 정보",
  "model": "claude-opus-4-8"
}
```
헤더: `Authorization: Bearer <API 키>` (키 입력 시)

**응답 (백엔드 → 앱, JSON)**
```json
{ "draft": "생성된 답안 초안 텍스트" }
```

> 백엔드에서 Claude API를 호출할 때, `guidance`를 근거로 답안을 쓰게 하고
> 다운로드된 **지침서/길라잡이 PDF를 근거 코퍼스(RAG)**로 함께 넣으면 "자료 근거 인용" 품질이 올라갑니다.
> API 키는 절대 프런트엔드에 하드코딩하지 말고 백엔드 환경변수로 두세요.

## 5. 확장: 나머지 분야도 구조화하기

현재 구조화된 4개 공통분야와 동일한 방식으로 나머지 전문분야(진단혈액·임상화학 등)를 추출하면,
PDF 보기 → 기준별 답안 작성 화면으로 자동 전환됩니다.

1. `../data/checklists/2026/<분야>.pdf` 를 페이지 이미지로 렌더링(PyMuPDF)
2. 페이지 이미지를 OCR(비전)하여 `data/checklists/2026_<분야>.json` 생성
   (스키마: `field, year, source, sections[].subsections[].items[]{code,type,question,points,guidance,page}`)
3. `data/index.json` 재생성 → 끝 (앱이 자동 인식)

> PDF 텍스트 레이어가 손상(임베디드 서브셋 폰트)되어 일반 텍스트 추출은 불가하며 **이미지 OCR**이 필요합니다.

## 6. 출처·저작권

점검표 및 PDF 원문의 저작권은 **재단법인 진단검사의학재단**에 있습니다.
본 앱은 회원 권한 내 자료를 인증 준비 목적으로 활용하기 위한 보조 도구이며, 원문 재배포 용도가 아닙니다.
